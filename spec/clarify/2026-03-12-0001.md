# 規格重新生成 - 從代碼推導

日期：2026-03-12
模式：自動推導（無用戶互動）

## 來源

所有規格從以下代碼推導而來：
- PHP 後端：118 個核心檔案（Applications, Contracts, Domains, Infrastructure, Shared）
- TypeScript 前端：76 個檔案（pages, components, utils, types）
- 既有規格檔案作為參考但完整重新生成

## 決策紀錄

### 1. Activity 流程拆分
將整體系統拆為兩個 Activity：
- **LINE報名活動流程**：涵蓋設定 → 活動查詢 → 推廣連結 → LIFF → Carousel → 報名 → 通知 → 自動審核
- **工作流自動化流程**：涵蓋規則建立 → 發布 → 實例化 → 節點執行 → 完成/失敗/等待

### 2. Feature 分類
依據代碼中的 domain boundary 分為 5 個子目錄：
- `settings/`：設定相關（3 個 feature）
- `activity/`：活動查詢（1 個 feature）
- `promo-link/`：推廣連結管理（2 個 feature）
- `line/`：LINE 整合（3 個 feature）
- `registration/`：報名相關（5 個 feature）
- `workflow/`：工作流引擎（6 個 feature）

### 3. API Endpoints
從代碼確認 5 個 REST endpoint：
- `GET /activities`
- `GET /options`
- `POST /options`
- `POST /revoke-google-oauth`
- `POST /liff`（public）
- `POST /line-callback`（public）

### 4. Entity Model
維持原有 4 個 CPT + 3 個 wp_options 結構，增加了約束說明和 Feature 對應關係。
