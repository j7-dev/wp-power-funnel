# Clarify Session 2026-03-26 00:00

## Idea

用戶希望將「自動化 > 觸發條件」的下拉選單改為動態從後端 API 取得資料，而非前端硬編碼。
觸發條件的值來自 `J7\PowerFunnel\Infrastructure\Repositories\WorkflowRule\Repository::get_trigger_points`，
這樣其他開發者可以透過 `apply_filters('power_funnel/workflow_rule/trigger_points', ...)` 這 hook 擴展觸發時機點。

## Q&A

- Q: 每個觸發點的回應格式應包含哪些欄位？ → A: 選項 A，只回傳 hook + name（直接對應前端 Select 的 value + label）
- Q: 此 API 是否需要接受任何查詢參數？ → A: 選項 A，無參數，直接回傳全部觸發點
